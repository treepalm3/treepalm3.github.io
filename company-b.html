Database & API

สคีมาหลัก + ตัวอย่างเอนด์พอยต์มาตรฐาน

Schema (ย่อ): 
users (id, name, email [unique], password_hash, role, created_at)
products (id, sku [unique], name, price, stock, created_at)
orders (id, user_id, total, status, created_at)
order_items (order_id, product_id, qty, price)

INDEX:
- users.email, products.sku, products.name, orders.created_at
FK:
- orders.user_id → users.id
- order_items.order_id → orders.id
- order_items.product_id → products.id
รองรับ Migration/Seeder และโหลดชุดข้อมูลตัวอย่างได้
REST API (ตัวอย่าง): 
POST /api/v1/auth/login
200 { "token":"<jwt>", "user":{"id":1,"name":"Admin","role":"admin"} }

GET /api/v1/products?page=1&limit=20&q=monitor
200 { "data":[...], "meta":{"page":1,"total":120} }

POST /api/v1/orders
Body: { "items":[{"product_id":1,"qty":2}] }
201 { "id":101, "total": 1200.00, "status":"new" }
มี Rate-Limit, Error Shape เดียวกันทั้งระบบ, รองรับ Swagger/OpenAPI

Security & DevOps

แนวทางที่ใช้จริงในงานโปรดักชันขนาดเล็ก-กลาง

Security Checklist

  • Password Hash: password_hash() (bcrypt/argon2)
  • Prepared Statements ทุกคำสั่ง SQL
  • CSRF Token ฟอร์ม + SameSite Cookie
  • Input Validation/Filtering ทั้งฝั่ง Client/Server
  • RBAC: Roles/Permissions แยกชัด
  • มีแผนเพิ่ม Content Security Policy

DevOps / Deploy

# docker-compose.yml (ย่อ)
services:
  nginx: { image: nginx:alpine, volumes: ["./app:/var/www/html"], ports: ["8080:80"] }
  php:   { image: php:8.2-fpm,   volumes: ["./app:/var/www/html"] }
  db:    { image: mysql:8, environment: { MYSQL_DATABASE: app, MYSQL_ROOT_PASSWORD: secret } }
  • แยก .env ตาม stage (dev/staging/prod)
  • สคริปต์ Backup DB รายวัน, Healthcheck เบื้องต้น
  • CI เบื้องต้นสำหรับ Lint/Test ก่อน deploy
